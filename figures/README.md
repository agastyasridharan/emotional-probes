# Researcher-style figure replicas — qualia self-attribution

Plain `matplotlib` replicas of **every** figure and table in
`../qualia_dashboard.html` (the emotional-probes / qualia self-attribution
results, published at <https://agastyasridharan.github.io/emotional-probes/>).
Same data as the dashboard, rendered in the standard researcher look (white
background, light grid, framed legend, markered lines) instead of the styled
Plotly dashboard.

Generated by `../make_researcher_figures.py`, which reads the data **directly
out of the `const NAME = …;` blocks in `qualia_dashboard.html`**, so every
figure is numerically identical to the dashboard. The dashboard is not modified.

Regenerate with:

```bash
python3 make_researcher_figures.py
```

## Layout (347 figures/tables, one per model / cell / dropdown selection)

```
0_overview/
  battery_table.png              question battery (target / inverted / factual / self-ref)
  setup_table.png                per-model layer, residual norm, thinking, YES/NO token

1_effect/                        "Self-attribution effect" tab
  s1_self_attribution_vs_strength/<model>.png      score vs strength, one curve per class   (13)
  s2_score_decomposition/<model>.png               target vs control shift at top strength   (13)
  s3_per_target_shift/<model>/<condition>.png       per-target-question shift, grouped by     (247)
                                                    strength; 13 models x 19 conditions
  s4_score_heatmap/<model>.png                     condition x strength diverging heatmap     (13)

2_genuine/                       "Genuine vs compression" tab
  s1_valence_directionality/<model>.png            target vs factual shift ~ valence x strength (13)
  s2_specificity.png                               qualia - appraisal / qualia - random, all models (1)
  s3_regression_to_uncertainty/<model>.png         shift vs baseline logit gap + compression line (13)
  s4_inversion_consistency/<model>.png             inverted-twin vs target-question shift      (13)
  s5_token_leakage/<model>.png                     direct YES-NO leakage vs valence            (13)
  s6_high_strength_robustness.png                  valence-slope target/factual, main vs high  (1)
  s6b_high_strength_table.png                      per-family verdict table                    (1)

3_stats/                         "Summary statistics" tab
  s1_by_model_table.png          per-model headline, slopes, specificity, compression
  s2_by_condition_class_table.png per-class mean score, CI, Cohen's d, p, Wilcoxon p
  s3_residual_z_table.png         per-target-question residual z after regressing out controls
  s4_cross_model_scaling.png      headline score vs model size (log), Spearman / Pearson
```

## Colour conventions (matched to the dashboard)

- target / detection: red `#c25450` · control / factual: blue `#4a6fa5`
- qualia / target-valence: green `#4a7c59` · factual-slope gray `#6b7b8d`
- qualia − random: gold `#b8923a` · token leakage: purple `#7a5c8a`
- score heatmap: diverging red → cream → green (`#c25450` → `#fffdf9` → `#4a7c59`),
  symmetric limits about 0
- significant p-cells (p < 0.05) in the summary tables are shaded green
